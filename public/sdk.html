2. Legacy API Key (Static Access)

Old-school method. **Limited to 50 RPM trial.** Use this for basic integration.

NODE.JS / FETCH SNIPPET
// Warning: Static keys are vulnerable to replay attacks.
const API_KEY = "sk_chaos_demo123"; // REPLACE WITH YOUR LIVE KEY
const userToken = "tk_sample_token_from_login";

fetch("/api/v1/external/verify", {
    method: "POST",
    headers: {
        "Content-Type": "application/json",
        "X-CHAOS-API-KEY": API_KEY // The key is static and visible
    },
    body: JSON.stringify({ token: userToken })
}).then(res => res.json()).then(data => console.log(data));

Quota limited and not AI-native.

3. Chaos Pulse (AI-Native Handshake)

**Recommended.** Dynamic, zero-touch verification. **Extended 500 RPM trial.**

FUTURE SDK IMPLEMENTATION
// This handles the BLS multi-signature and dynamic nonce generation.
import { ChaosAgent } from '@aplus/chaos-sdk';

const agent = new ChaosAgent({ endpoint: "/api/v1/pulse" });

// Zero configuration: Agent automatically fetches keys and rotates nonces.
async function verifyAgent() {
    const userToken = "tk_sample_token_from_login";
    const verification = await agent.verify(userToken);
    
    // Check if the temporal signature passed the DREAMS Protocol
    if (verification.assurance === 'HIGH') {
        console.log("ACCESS GRANTED: AI Trust Established.");
    }
}
verifyAgent();

No static keys required. Built for machine concurrency.
